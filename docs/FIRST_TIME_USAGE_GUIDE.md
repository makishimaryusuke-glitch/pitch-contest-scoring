# 初回運用ガイド

## 📋 現在の実装状況

初回運用に向けて、以下の改善が既に実装されています：

✅ **改善版プロンプト**
- 具体的なルーブリック（各スコア帯の基準）
- 評価のポイント（5項目のチェックリスト）
- Few-shot Learning用の評価例（高・中・低得点例）

✅ **温度設定の最適化**
- OpenAI: 0.1（一貫性重視）
- Gemini: 0.1（一貫性重視）

✅ **評価理由の詳細化**
- 200文字程度の詳細な説明を要求

## 🎯 初回運用での推奨事項

### 1. 採点前の準備

#### APIキーの設定
- OpenAIまたはGoogle GeminiのAPIキーを設定
- 「⚙️ API設定」ページで確認

#### 評価基準の確認
- 各評価項目のルーブリックを確認
- 審査員の方々にも評価基準を共有

### 2. 採点の実行

#### 推奨される運用フロー
1. **提出資料のアップロード**
   - 「📝 採点ワークフロー」ページで参加校を選択
   - テーマ情報とファイルを入力

2. **AI採点の実行**
   - 「🚀 AI採点を実行」ボタンをクリック
   - 採点結果を確認

3. **結果の確認**
   - 「🏫 参加校管理」ページでデータ一覧を確認
   - 各評価項目のスコアと評価理由を確認

### 3. 採点結果の活用

#### 審査員による確認
- AI採点結果を参考に、審査員が最終的な採点を決定
- AI採点と人的採点の差異を記録（今後の改善に活用）

#### データの保存
- 採点結果は自動的に保存されます
- CSV形式でエクスポートも可能

## 📊 今後のデータ蓄積に向けて

### 初回運用で収集すべきデータ

#### 1. 採点結果データ
- 各参加校の採点結果
- 各評価項目のスコア
- 評価理由

#### 2. 審査員による採点データ（推奨）
- 可能であれば、審査員にも同じ作品を採点してもらう
- AI採点と人的採点の比較データとして活用

#### 3. フィードバックデータ
- 審査員からのフィードバック
- 採点結果に対する意見や改善点

### データの保存方法

#### 自動保存
- 採点結果は`data/`ディレクトリに自動保存されます
- JSON形式で保存

#### 手動エクスポート
- 「🏫 参加校管理」ページからCSV形式でエクスポート可能
- バックアップとして保存推奨

### 次回以降の改善に向けて

#### データ蓄積後の活用
1. **評価例の収集**
   - 高得点作品の評価例を収集
   - プロンプトに追加して精度向上

2. **基準データセットの作成**
   - 複数審査員による採点データを収集
   - AI採点の精度検証に活用

3. **プロンプトの改善**
   - 実際の採点結果を分析
   - 必要に応じてプロンプトを調整

## ⚠️ 注意事項

### 初回運用での注意点

1. **AI採点は補助ツールとして使用**
   - AI採点結果を参考に、最終的な採点は審査員が決定
   - AI採点のみで採点を決定しない

2. **評価理由の確認**
   - 各評価項目の評価理由を必ず確認
   - 不適切な評価がないかチェック

3. **異常値の確認**
   - 極端に高い/低いスコアがないか確認
   - 必要に応じて再採点を検討

4. **データのバックアップ**
   - 採点結果は定期的にエクスポートしてバックアップ
   - データ損失を防ぐ

## 🔍 トラブルシューティング

### よくある問題と対処法

#### 1. 採点結果が期待と異なる場合
- 評価理由を確認して、どの点が評価されたか確認
- 必要に応じて再採点を実行

#### 2. APIキーのエラー
- 「⚙️ API設定」ページでAPIキーを確認
- 環境変数の設定を確認（Streamlit Cloudの場合）

#### 3. ファイルの読み込みエラー
- ファイル形式を確認（PDF、PowerPoint、テキスト）
- ファイルサイズを確認（大きすぎる場合は分割）

## 📝 運用チェックリスト

### 採点前
- [ ] APIキーが設定されているか確認
- [ ] 参加校が登録されているか確認
- [ ] 評価基準を確認

### 採点中
- [ ] 提出資料が正しくアップロードされているか確認
- [ ] 採点が正常に完了したか確認
- [ ] 評価理由を確認

### 採点後
- [ ] 採点結果を確認
- [ ] データをエクスポートしてバックアップ
- [ ] 審査員と結果を共有

## 💡 今後の改善に向けて

### データ蓄積の重要性

初回運用では過去データがなくても問題ありませんが、以下のデータを蓄積することで、将来的にAI採点の精度を大幅に向上させることができます：

1. **採点結果データ**
   - 各作品のスコアと評価理由
   - 審査員による採点との比較

2. **評価例データ**
   - 高得点作品の特徴
   - 低得点作品の特徴

3. **フィードバックデータ**

   - 改善点の提案

これらのデータを蓄積することで、次回以降のコンテストでは、より精度の高いAI採点が可能になります。
